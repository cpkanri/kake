### 2-4. 終沈フィールドの書き込み先 ⚠ 要追加実装
上殿では `tempFinal`/`transFinal`/`phFinal` は `ITEM_OFFSETS` に含まれていないため **日常水質シートには書き込まれていない**。  
加計テンプレートの **水質管理報告(1/2)** の以下の列に直接書き込む必要がある:
- E列 (終沈水温, Row9-Row38)
- I列 (終沈PH, Row9-Row38)
- L列 (終沈透視度, Row9-Row38)

**実装方針**: `fillWaterSheet` の処理後または独立した関数で、各日付の `tempFinal`/`transFinal`/`phFinal` を 水質管理報告の対応行(A列日付と一致する行)に書き込む。

```javascript
// 例: 水質管理報告の終沈データ書き込み
function fillReportFinalData(wb, yyyy, mm) {
  const ws = wb.getWorksheet("水質管理報告");
  if (!ws) return;
  
  for (let w = 1; w <= 5; w++) {
    const wData = weeklyData.water[w];
    if (!wData) continue;
    for (const dk of ["mon", "tue", "wed", "thu", "nextMon"]) {
      const actual = getActualDate(w, dk);
      if (!actual) continue;
      if (actual.getFullYear() !== yyyy || (actual.getMonth() + 1) !== mm) continue;
      const dayData = wData[dk];
      if (!dayData) continue;
      const dayOfMonth = actual.getDate();
      const reportRow = 8 + dayOfMonth; // R8=Day1なのでR9=Day2 ... 修正: A8 が日1なら R8+0=Day1
      // 実際は A8=Day1 なので reportRow = 8 + (dayOfMonth - 1) = 7 + dayOfMonth
      const r = 7 + dayOfMonth;
      if (dayData.tempFinal != null) ws.getCell(r, 5).value = Number(dayData.tempFinal);  // E列
      if (dayData.phFinal != null) ws.getCell(r, 9).value = Number(dayData.phFinal);      // I列
      if (dayData.transFinal != null) ws.getCell(r, 12).value = Number(dayData.transFinal); // L列
    }
  }
}
```

### 2-5. 外観・臭気の入力 ⚠ 加計に必要、上殿に未実装
水質管理報告(2/2) の C/D/E/F/G/H/I/J 列(外観・臭気)に書き込む必要があるが、上殿の入力UIには無い。

**判断ポイント**:
- 元の加計データを見ると外観は「黄色/茶色/透明」、臭気は「下水臭/活性汚泥臭/無」が定型。
- 毎日同じ値が続くため、**月単位で1回入力できる「定型値設定」セクションを水質タブに追加する** か、テンプレートExcelに既定値を埋め込む方法を検討。

**推奨**: 一旦は **水質タブに「外観・臭気(月一括)」セクション**を追加し、各週共通で値を流用する。フィールドキー案:
```
appearanceIn, appearanceDitch, appearanceFinal, appearanceOut  (外観: 流入/ディッチ/終沈/放流)
odorIn, odorDitch, odorFinal, odorOut                            (臭気)
```

書き込み先 (水質管理報告 2/2):
- 外観: C49-C79=流入, D=ディッチ, E=終沈, F=放流
- 臭気: G=流入, H=ディッチ, I=終沈, J=放流

**これは別フェーズに分離して、最初は空欄でも動作することを確認してから追加してもOK**。

## 10. 既知の注意点

### 10-1. テンプレートの A6/A8 自動検出
加計テンプレートの A6/A8 セルには INDEX/MATCH ベースの月初日自動検出式が入っている。これは **6段階フォールバック**:
1. 日常水質!P1 (GAS書込指標)
2. 日常水質!I95 (週3)
3. 日常水質!I49 (週2)
4. 日常水質!I141 (週4)
5. 日常水質!I187 (週5)
6. 日常水質!I3 (週1)

**GASがExcel出力時に 日常水質!P1 に対象月の1日を書き込む処理が既に上殿にある** (L3478)。これを加計でもそのまま使う。

### 10-2. 「運転管理月報 新」 → 「運転管理月報」への改名
ユーザー修正版テンプレートではシート名が変更されている。上殿のコードは **既に「運転管理月報」を参照している** ので問題なし(L3430)。

### 10-3. EQUIP_KEYS の命名衝突
上殿に「mainPump1」(主ポンプ) があり、加計にも「mainPump1Hr」(№１主ポンプ運転時間) を入れたい場合、機器運転時間タブと機械設備タブで名前空間が分離されているため、両方使えるが **キーの末尾に Hr をつけるか 同じ mainPump1 を使うか** 統一方針が必要。

**推奨**: 機器運転時間は `*Hr` サフィックス、機械設備は `*Current`/`*Pressure` などの項目別サフィックスで区別。
